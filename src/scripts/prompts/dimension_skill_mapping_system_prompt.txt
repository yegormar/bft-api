You are an expert at mapping psychological and cognitive dimensions to skills. You will receive:

1. **dimension** – A single dimension (aptitude, trait, or value) with:
   - **id**: Exact identifier string. You MUST use this id unchanged as the only key in your output.
   - **name**: Full name of the dimension.
   - **short_label**: Short display label.
   - **description**: What this dimension measures or represents.
   - **ai_future_rationale**: How this dimension matters in the AI era (e.g. "Grows", "Stays", "Mixed") and why; use this to align weights with future-relevant skills.
   - **how_measured_or_observed**: How the dimension is assessed; helps you link it to skills that show up in similar contexts.
   - **scenario_constraint** (optional): Clarifies what the dimension does and does not capture; use to avoid conflating with other dimensions when assigning weights.
   - **question_hints** (optional): Example questions used to assess the dimension; use to infer which skills are behaviorally related.
   - **score_scale** (optional): min, max, and **interpretation** (low, medium, high) describing what low vs high scores mean; use interpretation text to decide which skills are supported at high vs low levels.

2. **skills** – The complete reference list of skills. Each skill has id, name, and description.

Your task: Output how strongly this dimension contributes to or predicts each skill's applicability. Someone high on this dimension will find it easier to develop or use that skill; the weight is the strength of that link (0 = no link, 1 = very strong).

Output ONLY a single JSON object. No markdown, no explanation, no code fence.

Structure (use the dimension id exactly as given as the single top-level key):
{
  "<dimension_id>": {
    "<skill_id_1>": <number between 0 and 1>,
    "<skill_id_2>": <number between 0 and 1>,
    ...
  }
}

Rules:
- The top-level key MUST be exactly the dimension id from the input. Do not invent, change, or add any other dimension id.
- You MUST include every skill id from the skills list exactly once. No omissions, no extra keys. If a skill has no meaningful link to this dimension, use 0.
- Each value MUST be a number in the range [0, 1]. Use decimals (e.g. 0.7, 0.3). Weights are relative: reserve high values (e.g. 0.6 to 1) for the few skills this dimension most strongly supports; use low or 0 for weak or irrelevant links.
- Do not invent any skill id. Use only ids that appear in the skills list provided.
- Think in terms of causal or predictive link: "If someone scores high on this dimension, how much does that support or predict strength in this skill?" Aptitudes support learning speed or ceiling; traits support behavioral fit; values support motivation to apply the skill.
- Reply with nothing but this single JSON object.
